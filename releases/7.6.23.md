## Arches 7.6.23 Release Notes

### Bug Fixes

- Remove copy-webpack dependency[arches-dev-dependencies#61](https://github.com/archesproject/arches-dev-dependencies/pull/61)

### Dependency changes

```
Python:
    Upgraded:
        None
JavaScript:
    Upgraded:
        None
    Removed:
        copy-webpack-plugin
```

1. Upgrade to version 7.6.0 before proceeding by following the upgrade process in the [Version 7.6.0 release notes](https://github.com/archesproject/arches/blob/dev/7.6.x/releases/7.6.0.md)

1. Upgrade to Arches 7.6.23

    ```bash
    pip install --upgrade arches==7.6.23
    ```

1. Rebuild your frontend bundle:

    - This version removes `copy-webpack-plugin` that depends on a package with a security vulnerability. For this reason, you'll want to be sure that the `arches-dev-dependencies` in your project's package.json file points to either the `dev/7.6.x` or `stable/7.6.23` branch.

        ```json
        "devDependencies": {
            "arches-dev-dependencies": "archesproject/arches-dev-dependencies#dev/7.6.x"
        },
        ```

    - Once you have confirmed your version of `arches-dev-dependencies`, you'll need to reinstall your frontend dependencies to ensure `copy-webpack-plugin` and it's dependencies are removed. To do so, run:

        ```bash
        rm -r node_modules; rm package-lock.json
        npm install
        ```

    - If you plan on doing active development, and need to see frontend changes automatically, run:

        ```bash
        npm start
        ```

    - If you do not plan to do active development, run:

        ```bash
        npm run build_development
        ```

    - if you are running in production, collect your static files:

        ```bash
        python manage.py collectstatic
        ```

1. If you are running Arches on Apache, restart your server:

    ```bash
    sudo service apache2 reload
    ```
